## 二、未实现 ❌

### 🔥 第一梯队（核心生产力，差距最大）

#### 1. Diff 视图 ⭐⭐⭐
**优先级**：P0
**描述**：当工具调用 `write_file` / 编辑文件时，以 Git diff 形式展示代码变更（红绿对照），而不是 JSON 参数。
**对标**：Codex / Claude / Trae 都有
**实现思路**：
- 前端引入 diff 库（如 jsdiff）
- `tool.call_started` 时若 `tool_name === 'write_file'`，读取原文件内容
- `tool.call_finished` 时对比新旧内容渲染 diff
**依赖**：需要后端提供 `read_file` RPC 或在 tool 事件中附带 diff

```
期望效果：
┌──────────────────────────────────┐
│ write_file · foo.py              │
├──────────────────────────────────┤
│  def hello():                    │
│ -    print("hi")                 │
│ +    print("hello, world")       │
│ +    return None                 │
└──────────────────────────────────┘
```

#### 2. 文件树 / 代码预览 ⭐⭐⭐
**优先级**：P0
**描述**：左侧或抽屉式展示项目结构，点击文件可在 modal 中预览内容。
**对标**：Claude Code / Trae / Codex
**实现思路**：
- 新增 RPC：`fs.list_dir(path)` / `fs.read_file(path)`
- 前端新增文件树组件（可折叠目录）
- 点击文件弹出 modal 展示内容（支持语法高亮）

#### 3. 多模型切换 UI ⭐⭐
**优先级**：P1
**描述**：前端提供模型下拉菜单（deepseek / claude / gpt 等），允许用户切换。
**对标**：所有主流产品
**实现思路**：
- 后端新增 `model.list` RPC 返回可用模型
- 后端 `session.send_message` 支持 `model` 参数
- 前端顶栏新增下拉选择器

#### 4. 上下文附加（@ 引用） ⭐⭐⭐
**优先级**：P0
**描述**：输入框支持 `@文件路径` 或 `@会话` 引用，自动附加到消息。
**对标**：Claude / Trae / Codex
**实现思路**：
- 输入框检测 `@` 字符 → 弹出文件选择列表
- 选中后插入文件路径引用
- 发送时自动调用 `read_file` 附加文件内容

### 🟡 第二梯队（体验加分项）

#### 5. 消息编辑 / 重新生成
**优先级**：P1
**描述**：用户消息可编辑后重发；助手消息可"重新生成"。
**对标**：ChatGPT / Claude / Codex
**实现思路**：
- 用户消息气泡悬停显示 ✏ 编辑按钮
- 助手消息悬停显示 🔄 重新生成按钮
- 编辑后清空之后的所有消息并重发

#### 6. 代码块操作按钮
**优先级**：P1
**描述**：代码块右上角悬浮按钮：📋 复制、💾 应用到文件、🔄 重新生成。
**对标**：所有主流产品
**实现思路**：
- markdown.js 渲染代码块时插入按钮栏
- 复制：`navigator.clipboard.writeText`
- 应用到文件：解析代码块语言与文件名，调用 `write_file`

#### 7. 消息搜索 / 跳转
**优先级**：P2
**描述**：长会话中搜索关键字、跳转到某条消息。
**对标**：Claude / Codex（Cmd+K 搜索）
**实现思路**：
- 顶部搜索框或 ⌘K 快捷键
- 遍历 DOM 高亮匹配消息
- 上一个/下一个跳转

#### 8. 中断按钮 ⭐
**优先级**：P0
**描述**：流式输出时显示"停止"按钮，中断当前 run。
**对标**：所有主流产品
**实现思路**：
- 后端新增 `run.cancel(run_id)` RPC
- 前端 `_setSending(true)` 时把发送按钮变为"停止"
- 点击停止 → 调用 `run.cancel` → 后端 `self._running_runs` 取消对应 task

#### 9. 导出 / 分享
**优先级**：P2
**描述**：
- 导出会话为 Markdown / JSON
- 生成只读分享链接（需后端支持）
**对标**：Claude.ai（支持分享）

### 🟢 第三梯队（高级功能）

#### 10. Token / 成本仪表盘
**优先级**：P2
**描述**：顶部显示当前会话累计 token、估算成本、context window 占用可视化进度条。
**实现思路**：
- 累计所有 `llm.usage` 事件的 token 数
- 进度条可视化 context_pct
- 可配置模型单价估算成本

#### 11. 多 Tab 会话
**优先级**：P3
**描述**：浏览器多标签切换不同会话，互不干扰。
**实现思路**：
- 当前只能单 active session
- 需重构 session 管理逻辑为多实例
- 每个标签独立订阅事件

#### 12. 拖拽 / 粘贴上传图片
**优先级**：P2
**描述**：支持截图粘贴 → 作为图片消息发给多模态模型。
**实现思路**：
- 监听 `paste` / `drop` 事件
- 读取图片为 base64
- 发送 multimodal message（需后端支持）

#### 13. 会话重命名
**优先级**：P2
**描述**：侧边栏会话项支持重命名。
**对标**：设计文档已提到
**实现思路**：
- 后端 `session.rename` RPC（需新增）
- 前端会话项双击或点击 ✏ 按钮进入编辑模式

#### 14. 键盘快捷键 ⌘K 面板
**优先级**：P3
**描述**：全局命令面板（搜索会话、执行命令、切换主题）。
**对标**：Linear / VSCode
**实现思路**：
- ⌘K 快捷键打开顶部命令面板
- 支持模糊搜索
- 执行常用操作

#### 15. Agent 进度可视化
**优先级**：P3
**描述**：多步 ReAct 流程以步骤条形式展示。
**对标**：Codex / Trae
**实现思路**：
- 监听 `step.started` / `step.finished` 事件
- 顶部或侧边显示进度条：`Step 1 → Step 2 → Step 3`
- 每步可展开查看详情

## 五、核心差距总结

当前 RepoClaude Web 像一个**增强版的聊天界面**，但 Codex / Claude / Trae 已经进化成**集成开发工作台**。

**三大核心差距**：
1. **Diff 视图**（代码变更可视化）
2. **文件操作可视化**（文件树 + 代码预览）
3. **消息编辑**（编辑 / 重新生成 / 中断）

这三个能力决定了产品是"玩具"还是"生产级工具"。
